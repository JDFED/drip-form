## 配置
该字段配置在`schema`中：

```json title="schema" {12,13,14}
{
    "type": "object",
    "title": "上传组件",
    "ui": {
        "type": "uploader",
        "listType": "picture-card",
        "withCredentials": false,
        "canDrag": false,
        "accept": "https://www.jd.com",
        "action": "https://www.jd.com",
        "maxFileNumber": 5,
        "defaultFileList": true,
        "method": "get",
        "disabled": false
    },
    "fieldKey": "uploader_sDVWUt"
}
```

:::info 提示

组件具体配置依赖于所使用的UI库，DripForm仅作示例，具体可参考UI库对应文档。

:::

## API
| 参数        | 说明     | 类型     |  是否必填 | 默认值 |
|:-----------|:--------------------------------------------------------------|:--------|:---|:---|
| filedKey      | 表单数据存放字段（必须唯一）| string   | 必填 | - |
| options | 可选数据源              | options[ ] | 必填 | - |
| action      | 上传的地址 | string  | 必填 | - |
| accept      | 接受上传的文件类型 | string |  必填 | - |
| canDrag      |能否拖动  | boolean   | 非必填 | false |
| defaultFileList      | 默认已经上传的文件列表 | object[ ]   | 非必填 | - |
| disabled      | 值为 `true` 时，上传组件为禁用状态 | boolean   | 非必填 | false |
| listType      | 上传列表的内建样式，支持三种基本样式 `text`, `picture` 和 `picture-card` | string  | 非必填 | `text ` |
| maxFileNumber      | 限制上传数量。当为 1 时，始终用最新上传的文件代替当前文件 | number   | 非必填 | 1 |
| method      | 上传请求的 http method | string   | 必填 | Post |
| onChange      | 上传文件改变时的状态 | function   |  非必填 | - |
| withCredentials      | 上传请求时是否携带 cookie | boolean   | 非必填 | false |
